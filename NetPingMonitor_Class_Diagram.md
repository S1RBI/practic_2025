# Диаграмма классов приложения NetPingMonitor

## Введение
Данный документ описывает структуру классов Android-приложения NetPingMonitor для управления сетевыми устройствами NetPing. Система реализована с использованием архитектуры MVVM и современных Android-технологий.

## Ход работы
Система должна отражать следующие аспекты и отношения:

1. Приложение управляет несколькими сетевыми устройствами
2. Каждое устройство имеет набор реле, термостатов и логических правил
3. Пользователь может подключаться к устройствам и управлять их состоянием
4. Данные сохраняются локально через SharedPreferences
5. Сетевое взаимодействие реализовано через OkHttp
6. DI-контейнер управляет зависимостями

Классы должны включать следующие элементы:

### MainActivity
- Контекст приложения
- UI-компоненты (Compose)
- ViewModel
- Обработчики событий UI

### NetPingViewModel
- Состояние UI (LiveData/StateFlow)
- Репозиторий данных
- Методы управления устройствами:
  - `connectToDevice()`
  - `refreshData()`
  - `saveSettings()`
- Логика обновления состояния

### NetPingRepository
- Клиент OkHttp
- Кэшированные данные устройств
- Методы API:
  - `fetchDeviceData()`
  - `controlRelay()`
  - `updateLogicRules()`
- Парсеры ответов

### SavedDevice (Модель данных)
- ID устройства
- IP-адрес
- Учетные данные
- Состояние подключения
- Последнее время обновления

### NetPingDeviceData (Модель данных)
- Информация об устройстве
- Состояние реле
- Настройки термостатов
- Логические правила
- Статус сети

### NetworkModule (DI)
- Пул соединений
- Таймауты
- Интерцепторы
- Поставщик OkHttpClient

### Constants
- Таймауты сети
- Ключи SharedPreferences
- Коды ошибок

## Заключение
Диаграмма классов демонстрирует четкое разделение ответственности между компонентами:
- **Presentation Layer**: MainActivity + ViewModel
- **Domain Layer**: Repository
- **Data Layer**: Модели данных + NetworkModule

Система использует современные подходы Android-разработки, включая Jetpack Compose для UI, Hilt для DI и OkHttp для сетевых операций. Архитектура позволяет легко расширять функционал управления устройствами NetPing.
